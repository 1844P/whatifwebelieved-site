# Contributing to WhatIfWeBelieved

Thank you for considering contributing to the WhatIfWeBelieved website! We welcome contributions from the community to help improve this global faith and theology resource.

## How to Contribute

There are several ways you can contribute to this project:

### Reporting Issues
If you find a bug or have a suggestion for improvement, please:
1. Check if the issue has already been reported
2. Create a clear and descriptive issue report
3. Include steps to reproduce the issue if applicable
4. Add relevant screenshots or screen recordings if helpful

### Suggesting Features
We welcome feature suggestions! Please:
1. Clearly describe the feature and its purpose
2. Explain how it would benefit users
3. Provide any relevant examples or mockups if applicable

### Contributing Code
If you'd like to contribute code changes:

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes following our coding standards
4. Test your changes thoroughly
5. Submit a pull request with a clear description of your changes

## Development Setup

To set up the development environment:

1. Install Node.js (v18 or later recommended)
2. Clone the repository
3. Install dependencies:
   ```bash
   npm install
   ```
4. Start the development server:
   ```bash
   npm run dev
   ```
5. Open your browser to http://localhost:3000

## Coding Standards

Please follow these guidelines when contributing code:

### HTML
- Use semantic HTML5 elements
- Include proper alt text for all meaningful images
- Ensure proper heading hierarchy (h1-h6)
- Use semantic landmark elements (nav, main, section, etc.)

### CSS
- Follow the existing CSS variable naming conventions
- Use CSS custom properties (CSS variables) for colors, spacing, etc.
- Maintain mobile-first responsive design approach
- Keep CSS modular and reusable

### JavaScript
- Use modern ES6+ syntax
- Follow consistent code formatting
- Comment complex logic
- Avoid global variables when possible

### Accessibility
All contributions should maintain or improve accessibility:
- Ensure proper color contrast ratios (WCAG AA minimum)
- Include descriptive alt text for images
- Ensure keyboard navigability
- Use ARIA attributes appropriately when needed
- Test with screen readers when making significant changes

### Performance
- Optimize images before adding
- Minimize external dependencies
- Consider lazy loading for below-the-fold content
- Keep bundle sizes reasonable

## Pull Request Process

1. Ensure your code passes any existing tests
2. Update documentation as needed
3. Make sure your code follows the project's style guidelines
4. Write a clear, descriptive pull request title
5. Provide a detailed description of your changes
6. Reference any related issues in your pull request
7. Wait for maintainer review and address any feedback

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

## Questions?

If you have any questions about contributing, please don't hesitate to reach out through the project's issue tracker.

Thank you for helping make WhatIfWeBelieved better!
